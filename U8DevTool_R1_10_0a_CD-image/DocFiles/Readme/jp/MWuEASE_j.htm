3. 更新内容

今回のリリースでは、以下の内容が更新されています。

  • MWuEASE V2.20

・ チェックサム機能を追加しました。

 

以降は過去にリリースされたバージョンの更新内容です。

  • MWuEASE V2.10

・ コピーライト表記を新会社名（LAPIS Semiconductor Co., Ltd.）に変更しました。

 

  • MWuEASE V2.00

(1)簡易モードを追加しました。簡易モードでは、1ボタンをクリックするだけでターゲットLSIへの書き込みができます。

　　メインダイアログのモード切り替えは、 [View] メニューから行います。

 

(2) シリアル番号の長さを256バイトまで指定できるよう拡張しました。

[Serialization]ダイアログ上で指定するシリアル番号の長さを、256バイトまで指定できるように拡張しています。

 

(3) ターゲット情報の読み込み先のファイルを変更しました。

ターゲット情報の読み込み先のファイルを、FWuEASE.ini からTRGファイルに変更しました。

 

  • MWuEASE V1.10

(1) プログラム書き込み時にシリアル番号を付加して書き込む機能を新規に追加しました。

 

(2) 以下の不具合を改修しました。

- Connection List上で次のいずれかの条件を満たすターゲットに対して、ポップアップメニューの”Save Memory”が有効になってしまう。
このとき、”Save Memory”を選択しても、フラッシュメモリの内容が正しくファイルに保存されない。
    a. Statusが”Verify Error”になっている。
    b. Enableのチェックボックスがチェックされていない。
※本リリースバージョンでは、上記の場合ポップアップメニューの”Save Memory”が無効になります。

 

  • MWuEASE V1.01

・ 以下の不具合を改修しました。
- セキュリティID確認ダイアログにて[Initialize Flash memory and security setting]を選択して[OK]ボタンを押した場合、それまでにエラーが発生しているターゲットがあったとしても、そのターゲットに対応する[Connection List]の[Status]列の表示がPlug OKになってしまう。

 

  • MWuEASE V1.00

初版リリース

 

4. 制限事項

現バージョンのMWuEASEには、以下の制限事項があります。

(1)  Plug/UnPlug/Erase Memory/Write/Verifyの各処理の実行中においても、メインダイアログ右上の[×]ボタンが有効になります。
上記処理の実行中にメインダイアログ右上の[×]ボタンを押すと、終了確認ダイアログが表示されます。
この確認ダイアログでOKを選択すると処理の途中であってもMWuEASEは強制終了します。
