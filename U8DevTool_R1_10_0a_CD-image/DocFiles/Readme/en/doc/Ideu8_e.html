4. Restrictions

The current version of IDEU8 integrated development environment has the following restriction:

(1) Common use of the project file for IDEU8 by a 64-bit version of Windows and a 32-bit version of Windows
When sharing the project file for IDEU8 by a 64-bit version of Windows and a 32-bit version of Windows, since the file in system folders, such as a start-up file, is not found, an error may happen. Copy such a file to the folder of a project file, and add the file to a project.

 

(2) Notes of the account name when log on
Don't log on using the account name containing '&'.

 

(3) The "Edit| Find next" menu may not be done.
Retrieval processing may not be done even if the “Edit| Find next” menu (or F3 key) is selected. In that case, please select the menu again.

 

(4) The shortcut by a key of ten key cannot be used.
The shortcut by a key of ten key cannot be used. Please use keys other than ten key.

 

(5) It is not possible to specify the response file into the “Additional options field of the Compiler and assembler command line options” dialog.
It is not possible to specify the response file (@filename) into the “Additional options field” of the “General” tab of the “Compiler and assembler command line options” dialog. When the response is specified into the “Additional options” field, the CCU8 compiler or RASU8 assembler will issue an error.
Use the Command-line environment when you compile or assemble by using the response file.

 

(6) The command "Undo" currently doesn't work well.
This command is expected to cancel an input operation for the editor windows, however it isn't able to cancel the operation.

 

(7) The "Project| Debug" menu may not be done.
DTU8 debugger may not be launched even if the “Project| Debug” menu is selected. In that case, please select the menu again.

 

(8) Please put neither a project file nor a source file on the folder containing a space or a double-byte character in 64-bit Windows.
In the 64-bit version of Windows Vista or the Windows7, if a project file and a source file are put on the folder containing a space or a double-byte character, it may be unable to build correctly. Please be sure to put them on the folder into which neither a space nor a double-byte character goes.

 

(9) Please change the folder of a system path from a default folder in 64-bit Windows.
When IDEU8 is launched for the first time, it is necessary to set a system path. In that case, in the 64-bit version of Windows Vista or the Windows7, since the folder in which U8 development tools is installed, and a different folder will be specified if the folder name currently displayed by the default folder is set up directly, it cannot build. Please be sure to specify the folder for 32-bit applications (C:\Program Files(x86)\U8Dev\xxx) in which U8 development tools is installed.

 

(10) Please specify the existing folder as the output directory of the “compiler and assembler command line options”.
Building is not done when the folder not existing is specified.

 

(11) The color of a block comment does not change.
Even if it validates color construction, the color of a block comment does not change.

 

5. Important Notes

The current version of IDEU8 can read any project file (.PID) saved by an older version of the IDEU8.
However, it is impossible for an older version of the IDEU8 to read project files saved by the current version of the IDEU8.
