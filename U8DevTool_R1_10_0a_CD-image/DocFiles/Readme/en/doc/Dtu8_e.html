2. Supplement to the use of emulator

Observe the following when using the DTU8 debugger in connection with an emulator.

Note:
The method of, and the general notes on, using an emulator are described in hardware related documents (such as user's manuals). Refer to them as required.

2.1 Startup sequence

(1) Verify your installation

After you install the DTU8 debugger by the setup program of the U8 development tool, check to see if a group named "nX-U8" has been created in the "U8 Tools" folder accessible from the Windows Start menu > "All Programs."

(2) Start the emulator

Turn the emulator power on.

(3) Start the DTU8 debugger

From the Start menu, select "All Programs" > "U8 Tools" > "nX-U8" > "DTU8 debugger" icon to start the DTU8 debugger.

(4) Connect the DTU8 debugger and the emulator

Choose the emulator to be used from the "Target ICE" drop down list on the "Communications parameters" dialog box that the DTU8 debugger displays, then click the "OK" button.

2.2 Stop sequence

Do not turn the emulator power off before stopping the DTU8 debugger. This is because communication will be made with the emulator before the DTU8 debugger is stopped.

5. Restrictions

The current version of the DTU8 debugger has the following restrictions:

 

(1) DTU8 cannot recognize change of the code memory which does not use an on-chip debug function like a flash memory rewrite function.

DTU8 cannot recognize change of the code memory which does not use an on-chip debug function like a flash memory rewrite function of LSI. If the contents of a code memory are displayed or saved to a program file after rewriting, please first validate “Update code memory window” option on “Other” tab of the “Environment settings”, and then update the contents of a code memory by use “Update” on Window menu. Moreover, please validate the option “Auto Update when a break occur” concurrently with the above-mentioned setting to update the contents of the code memory automatically when a break occurs.

In addition, when a code memory is updated, the mnemonic corresponding to the contents of a memory after rewriting in the disassembly display of the trace window is displayed to the trace data before rewriting.

 

(2) A real-time tab is not switched to in the emulation execution mode in Dr.U16ICE mode.

The display of a watch window cannot be switched to a real-time tab from other tabs in the emulation execution mode in Dr.U16ICE. Please change a display to a real-time tab before an emulation start to refer to the value of a real-time tab in the emulation execution mode.

By turning ON the “Activate realtime-tab of watch window when executing program” option on “Other” tab of the “Environment settings”, the real-time tab is automatically displayed at the time of an emulation start.

 

(3) The [Calc.] button can certainly be pushed when the value of coverage is referred to of the status window in Dr.U16ICE mode.

When the value of the coverage of the status window is referred to after break, please be sure to push the [Calc.] button and to update the value of coverage.

 

(4) A “Fill...” and “Change...” menus are invalid to the memory of the segment 8 and higher in Dr.U8ICE, Dr.U16ICE, uEASE, nanoEASE mode.

A “Fill...” and “Change...” menus are invalid to the memory of the segment 8 and higher in Dr.U8ICE, Dr.U16ICE, uEASE, nanoEASE mode. Please rewrite the memory of the segments 0-7 which are those references origin.

 

(5) When an interrupt occurs in a step over, a break occurs at the head of an interruption routine in Dr.U16ICE, Dr.U8ICE, uEASE, nanoEASE mode.

When an interrupt occurs in a step over, a break occurs at the head of the interruption routine instead of the next instruction of the instruction by which the step was started.

 

(6) The option setting "Reset system" when loading a program file may be off.

If loading of a program file fails, the “Reset system” option in "load program file" dialog indicated by “File| load program file…” menu will be unchecked. Please check the "Reset system" check box again and load a program file to reset after a program file load.

 

(7) In an initial state, when a program file is saved, a public symbol is not saved.

The “Save public symbols” checkbox button on the “Save program file” dialog indicated by selecting the "File| Save program to files…" menu is unchecked in an initial state. Please check the "Save public symbols" check box and save a program file to save public symbols to a program file.

 

(8) It is possible that the debugger cannot reference variables correctly because of the effect by compiler optimization.

If, during the course of CCU8 compiler optimization, a local variable is assigned to a register, the debugging information consistency may be lost; as a result, it becomes impossible for the DTU8 debugger to reference variables correctly.

 

(9) If a transition is made to HALT/STOP mode during simulation mode of the DTU8 debugger, automatic generation of interrupt is disabled.

If a transition is made to HALT/STOP mode during simulation mode of the DTU8 debugger, automatic generation of interrupt is disabled. This is because, if transitioned to HALT/STOP mode, no instruction will be executed and, as a result, the cycle count will no longer be incremented.

 

To avoid this restriction, take either of the following measures:

Measure 1: Cancel the HALT/STOP mode either by making an interrupt occur by clicking an interrupt source button or by executing Step in or Step over.

Measure 2: Modify the customer's program so that control will not transition to HALT/STOP mode.

 

(10) The jump button on the Symbols dialog does not run correctly for the bit type symbols.

The jump button on the Symbols dialog indicated by selecting the "Tools| Symbols" menu does not run correctly for the bit type symbols.

 

(11) When the following operations are done for the replicated windows, the replicated windows may be overlapped to one.

Operation 1: Minimize the window and move the minimized window by dragging.

Operation 2: Restore the window and replicate the window by selecting the " Windows | Replicate" menu.

Operation 3: Minimize the replicated windows.

 

(12) During step over emulation, the " Window | Close active window" menu may be unavailable.

During step over emulation, if only a source window is displayed, the "Window | Close active window" menu may be grayed out and unavailable. Therefore the source window may not be closed.

 

(13) In the target LSI which has only the code memory of the physical segment 0 and has the data memory (the ROM reference area) of the physical segment 1 and higher, DTU8 saves the data which include the data of the physical segment 1 and higher to the file when you save the file on the " Save program to file" dialog.

DTU8 may issue an error when you specify the saved file which include the data of the physical segment 1 and higher in the "Load program file" dialog. To avoid this error, specify the address range (only physical segment 0) in the "Save address range "field of the "Save program file" dialog.

 

(14) If a communication error for the target happens when the contents of the memory are viewed, some incorrect values are shown in each window.

If a communication error for the target happens when the contents of the memory are viewed by either of the following operation, some incorrect values are shown in each window, without showing "**" mark which indicates the error.

- Execute "Jump" in the context menu of the code memory window, data memory window or memory window of physical segments 1 and higher.

- Add items to the watch window.

 

(15) The displayed return value of a function whose type is "struct", "union" or "enum" is wrong.

When the watch window or WATCH script command is used, the displayed return value of a function whose type is "struct", "union" or "enum" is wrong.

 

(16) When the local variables assigned to registers by WATCH script command, the wrong value is displayed.

When displaying local variables assigned to some general registers by WATCH script command, the wrong values are shown in the "Address" column instead of register names.

 

(17) About the frequency control register (FCON0,1) in case of uEASE use.

a)      Do not write the program made into the clock mode which is not mounted for the OSCM0/OSCM1 bit of FCON0 in the flash memory of target LSI. When this program is written in and the emulation is executed, the operation about execution becomes impossible, because the clock is not supplied to CPU. And DTU8+uEASE cannot be started by target LSI which wrote in this program. In these cases, Error number: 6300H is displayed. When you write in this program and the error occurs, please erase or change the flash memory by FWuEASE+uEASE.

 

b)      The OSCM0/OSCM1 bit of FCON0 cannot be changed in the SFR window on DTU8.

 

c)      When ENOSC and SYSCLK of FCON1 are set to 1, even if ENOSC is set to 0 in the SFR window on DTU8, the display of SYSCLK is still 1. However, SYSCLK inside target LSI is 0. Therefore, the emulation is executed in the state of SYSCLK=0 as it was operated in the SFR window.

 

(18) When changing data is specified the range address in the Change data memory dialog on the Disassembly window and change it, DTU8 may not change it correctly.

When you change the data of code memory or memory in physical segments 1 and higher, please use the Code memory window or Memory in physical segments 1 and higher window.

 

 

6. Important Notes

(1) The current version of the DTU8 debugger can read any project file saved by an older version of the debugger.
However, it is impossible for an older version of the DTU8 debugger to read project files saved by the current version of the debugger.

(2) Do not write program code or data to test area. If it is written to test area, DTU8 may not run correctly.

(3) Do not operate DTU8 debugger except a regular procedure. If it is operated except a regular procedure, DTU8 may not run correctly.
