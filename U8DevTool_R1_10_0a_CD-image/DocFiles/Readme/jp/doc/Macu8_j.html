2. DCLファイル

MACU8アセンブラパッケージは、nX-U8/100をコアにもつすべての機種に対応します。

これは、機種ごとに異なる部分に対する定義を、各ソフトウェアの実行時に外部から与えることによって実現しているからです。異なる部分は、メモリ構成、SFRの名前と属性、および使用可能な命令などです。これらの定義を含むファイルがDCLファイルです。１つの機種には１つのDCLファイルが用意されています。

DCLファイルは、アセンブラによって読み込まれます。その他のソフトウェアはアセンブラが作成したオブジェクトファイルを通じて、DCLファイルで定義されている情報のうち、処理に必要なものを受け取ります。

DCLファイルは、ソフトウェアによって使用されるだけでなく、ドキュメントとしての役割も持ち合わせています。たとえば、プログラムで使用できるSFRの名前は、DCLファイルを読むことによって知ることができます。DCLファイルの読み方については、DCLU8_J.HTMLファイルを参照してください。DCLファイルはテキストファイルなので、画面やプリンタなどに出力することができます。

DCLファイルは、ソフトウェアの処理を制御するための重要なファイルですので、内容を書き換えないでください。

3. 更新内容

今回のリリースでは、RLU8およびRASU8の内容を更新しています。

■    RLU8

  • RLU8 V1.51

(1)   関数・テーブルが参照されているにも関わらずリンク対象外と扱う場合がある問題を改修しました。

(2)   割付可能なROMが#0にだけしか存在しない場合に、ROM Window領域に空きがあるにも関わらずTABLEセグメントが割り付けられない問題を改修しました。これに伴い、TABLEセグメントの優先度を変更しています。V1.50以前と同じ優先度で割り付ける場合には、新規に追加した/CPオプションを指定してください。

(3)   リンク時にエラーを検出したときに、アプリケーションエラーが発生し、強制終了する問題を改修しました。

以下は過去にリリースされたバージョンの更新内容です。

  • RLU8 V1.50

(1)   関数・テーブルの参照関係をチェックし、参照されない関数・テーブルをリンクしない機能を追加しました。
この機能を使用するには、CCU8 V3.30、RASU8 V1.60およびRLU8 V1.50が必要です。
デフォルトの機能として動作するため、オプションの指定は不要です。

(2)   /LAオプションを追加しました。
関数・テーブルの参照関係をチェックせずに、すべての関数・テーブルをリンクするには、/LAオプションを指定します。

  • RLU8 V1.40

  • RLU8 V1.20

(1)   コンパイラのバージョン情報をマップファイルに出力するように変更いたしました。コンパイラのバージョン情報は、アセンブラのバージョン情報と並んで出力されます。コンパイラのバージョン情報がオブジェクトファイルに含まれていない場合、マップファイルには”-.- -”が出力されます。

4. エラーファイルによるタグジャンプ

一部のエディタは、現在カーソルが置かれている行にあるファイル名と行番号の並びから自動的にそのファイルをオープンし、その行を表示する機能を持っています。RASU8が作成するエラーファイルを使って、この機能を利用することができます。この機能についての詳細は、使用するエディタのマニュアルを参照してください。

6. 制限事項

現バージョンのRLU8には、以下の制限事項があります。

■    RLU8

    変数を絶対番地に割り付けている場合に、インターナルエラーが発生する場合があります。

[発生条件]

@ absoluteプラグマ、segnoinitプラグマ、またはseginitプラグマのいずれかにて変数を絶対番地に割り付けている。

A 以下のいずれかの記述が上記@と同じファイルに存在する。

(a) interruptプラグマにて割り込み関数を定義している。

(b) swiプラグマにてソフトウェア割り込み関数を定義している。

(c) segconstプラグマにてconst変数を絶対番地に割り付けており、その変数に対する初期値として別のconst変数や関数のアドレスを与えている。

　　例えば、以下のような記述です。

static void test_func(void) { }

#pragma segconst 0x8000

void (* const fp)() = test_func;

 

[発生例]

unsigned char skip;

#pragma absolute skip 0x0E000

static void int_nmi(void);

#pragma interrupt int_nmi 0x0a 1

static void int_nmi(void)

{

}

 

[回避策]

以下のいずれかの方法で回避してください。

回避策@：segnoinitプラグマまたはseginitプラグマを用いて名前付きで定義したセグメントに変数を割り付け、リンク時にアドレスに割り付けるようにする。

以下に例を示します。

手順１：上記の発生例のソースファイルを以下のように修正します。

#pragma segnoinit "DATA_0E000"

unsigned char skip;

#pragma segnoinit

手順２：IDEU8の[ターゲットオプション]の[セグメント]タブにて、「優先的に割り付けるDATAセグメント」をチェックし、そのフィールドに以下のように指定してください。

DATA_0E000-0E000h

 

回避策A：/LAオプションを指定する。

IDEU8の[ターゲットオプション]の[一般]タブにて、「追加オプション」フィールドに /LA を指定してください。ただし、このオプションを指定した場合、参照されない関数やテーブルもリンクされます。
