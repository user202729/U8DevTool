4. 制限事項

現バージョンのIDEU8には、以下の制限事項があります。

(1) 32bit版Windowsと64bit版Windowsで同一のプロジェクトファイルを使用する場合に、登録したファイルが見つからずエラーとなる場合があります

32bit版Windowsで作成したIDEU8用プロジェクトファイルを64bit版Windows上のIDEU8で開いたとき、または64bit版Windowsで作成したIDEU8用プロジェクトファイルを32bit版Windows上のIDEU8で開いたときに、U8開発環境がインストールされたフォルダが異なるためプロジェクトに登録したスタートアップファイルやシステムヘッダファイルが見つからずエラーとなることがあります。
このような場合、エラーとなったファイルをU8開発環境がインストールされているフォルダ以外のフォルダ（プロジェクトフォルダなど）にコピーし、コピーしたファイルをプロジェクトに登録してください。

(2)’&’を含むアカウント名でログオンした場合、正常にビルドできません

以下の文字を含むアカウント名でログオンしている場合、正常にプログラムのビルドができない場合があります。この文字を含まないアカウント名でログインしてビルドを行ってください。
  &

(3) [次を検索]が実行されないことがあります

検索時に「次を検索」メニュー（またはF3キー）を実行しても、検索処理が行われないことがあります。その場合は再度メニューを実行してください。

(4) テンキーを使用したショートカットは使用できません

キーボードのテンキーにある数字を使用してメニューのショートカットを行うことはできません。テンキー以外にあるキーを使用してください。

(5) [コンパイル/アセンブル オプション]の[コンパイラ制御]および[アセンブラ制御]の[追加オプション]フィールドには応答ファイルを指定できません

[コンパイル/アセンブル オプション]ダイアログの[一般]タブにある、[コンパイラ制御]および[アセンブラ制御]の[追加オプション]フィールドには応答ファイル(@filename)を指定することはできません。応答ファイルを指定した場合、コンパイル/アセンブル時にエラーが出力されます。
応答ファイルを使用してコンパイル/アセンブルを行う場合には、Command-line環境をお使いください。

(6) [元に戻す]は動作しません

内蔵エディタへの入力操作を取り消す機能である[元に戻す]は動作しないため、入力操作を取り消すことはできません。

(7) [デバッグ]が実行されないことがあります

「プロジェクト」メニューの「デバッグ」を実行しても、DTU8デバッガが起動されないことがあります。その場合は、再度「プロジェクト」メニューの「デバッグ」を実行してください。

(8) 64ビットWindowsでは、スペースや全角文字の入ったフォルダにプロジェクトファイルやソースファイルを置かないでください

Windows Vistaおよび7の64ビット版では、スペースや全角文字の入ったフォルダにプロジェクトファイルやソースファイルを置くと、正常にビルドできないことがあります。必ずスペースや全角文字の入らないフォルダに置いてください。

(9) 64ビットWindowsでは、システムパスのフォルダをデフォルトから変更してください

初めてIDEU8を起動する際にシステムパスを設定する必要があります。その際、Windows Vistaおよび7の64ビット版では、デフォルトで表示されているフォルダ名をそのまま設定するとU8開発環境がインストールされているフォルダと異なるフォルダが指定されるためビルドが行えません。必ずU8開発環境がインストールされている32ビットアプリケーション用フォルダ（C:\Program Files(x86)\U8Dev\xxx）を指定してください。

(10) [コンパイル/アセンブル オプション]の[ディレクトリ]には、存在するフォルダを指定してください

存在しないフォルダを指定すると、正常にビルドが行われません。

(11) カラー構文を有効にしてもブロックコメントの色は変わりません

ソースウィンドウでカラー構文を有効にしてもブロックコメントの色は変わりません。

 

5. 注意事項

現バージョンのIDEU8では、現バージョン以前のIDEU8で保存したプロジェクトファイル(.PID)を読み込むことができます。
ただし、現バージョンのIDEU8で保存されたプロジェクトファイルを、現バージョンよりも前のIDEU8で読み込むことはできませんので、注意してください。
