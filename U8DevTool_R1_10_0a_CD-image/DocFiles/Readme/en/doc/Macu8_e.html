2. DCL files

The MACU8 assembler package supports all models which has nX-U8/100 core by isolating such particulars as memory structure, SFR names and attributes, and instructions available in separate DCL files. Components are thus able to automatically tailor their operation to the target device by making reference to the appropriate external file.

Only the assembler reads in the DCL file. The other components obtain the necessary DCL file information from the object files created by the assembler.

DCL files, in addition to the above software role, also serve to document for the developer such model particulars as the names of special function registers (SFRs) available to programs. For help in interpreting their contents, refer to the DCLU8_E.HTML file. DCL files are text files, so can be displayed on the screen or sent to the printer.

Because of their importance in controlling software operation, do not modify the contents of DCL files.

4. Tag jumps using error file

RASU8 creates error files in a format compatible with text editors capable of reading the file name and line number from the beginning of the current line, opening the specified file, and automatically jumping to that line. Consult your editor manual for further details on using this handy function.

6. Restrictions

The current version of the RLU8 linker has the following restriction:

RLU8
    An internal error may occur to description which assigns the variable to absolute address by using absolute, segnoinit or seginit pragma.

[Occurrence Conditions]

This problem occurs when the following conditions are satisfied.

(a) The variable is assigned to absolute address by using absolute pragma, segnoinit pragma or seginit pragma.

(b) Either description of the followings exists in the same file as (a).

(i)  The interrupt function is defined by interrupt pragma.

(ii)  The software function is defined by swi pragma.

(iii) The const variable is assigned to absolute address by segconst pragma. And as an initial value for the const variable, the address of another const variable or function is given. For example, it shows the following description.

static void test_func(void) { }

#pragma segconst 0x8000

void (* const fp)() = test_func;

 

[Occurrence Example]

unsigned char skip;

#pragma absolute skip 0x0E000

static void int_nmi(void);

#pragma interrupt int_nmi 0x0a 1

static void int_nmi(void)

{

}

 

[Workaround]

Please avoid by either method of the followings.

(a) Define the named segment by using segnoinit pragma or seginit pragma, and assign the variable to the named segment.

Then, specify address by linker option. An example is shown below.

Step 1: The description of above example is modified as follows.

#pragma segnoinit "DATA_0E000"

unsigned char skip;

#pragma segnoinit

Step 2: Please specify like  DATA_0E000-0E000h  in the “Segments” field of DATA of [Target options][Segment] of IDEU8.

 

(b) Specify /LA option.

Please specify  /LA  in the “Additional options” field of [Target options][General] of IDEU8.

Please note that when this option is specified, the function and table which are not referred to are also linked.
