3. Updated Contents

The following updates have been added in this release:

  • MWuEASE V2.20

The Checksum function has been added.

 

The following updates were added in previous release:

 

  • MWuEASE V2.10

The copyright has been changed to new company name (LAPIS Semiconductor Co., Ltd.)

 

  • MWuEASE V2.00

(1)   A simple mode has been added. On the simple mode, it is able to program to flash memory of the target microcontroller by clicking one button.

      To change the mode of a dialog, select the mode from [view] menu.

 

(2) The number of bytes of the serial number has been extended to 256 bytes.
It is able to extend to 256 bytes for the number of bytes of the “Serialization” dialog.

(3) The target information file for MWuEASE has been changed.
The target information file for MWuEASE has been changed from FWuEASE.ini to TRG file.

 

  • MWuEASE V1.10

(1)   New feature that writes a program code with a serial number is added.

 

(2) The following problem has been fixed.
- On the Connection List, “Save Memory” pop-up menu for the target, which meets either of the following conditions, is enabled. In this case, even if “Save Memory” is selected, the content of the flash memory is not saved into a file correctly.
    a. [Status] shows “Verify Error”.
    b. [Enable] checkbox is not checked.
 Note: In this release version, “Save Memory” pop-up menu is disabled in the above case.

 

  • MWuEASE V1.01

The following problem has been fixed.
- When [Initialize Flash memory and security setting] in the security ID checking dialog box is selected and the [OK] button is pushed, even if there is a target in which an error has been occurred, “Plug OK” is displayed for that target in the [Status] column of the [Connection List].

 

  • MWuEASE V1.00

Initial release

 

4. Restrictions

The current version of MWuEASE has the following restriction:

(1)   The close button [x] at the upper right corner of the main dialog is enabled while Plug/UnPlug/Erase Memory/Write/Verify processing
If the close button [x] is clicked while these processing is just executed, the confirmation dialog is displayed.
MWuEASE is forced to close by clicking OK button in the confirmation dialog, even if the processing is not yet finished.
